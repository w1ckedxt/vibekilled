## Wat is deze sessie gedaan

### 1. Instant card — optimistic drop (geen netwerk meer in de weg)
**Root cause van "TE LAAT":** de card wachtte op de `await createPin(...)`-round-trip
vóór 'ie überhaupt verscheen (modal bleef open met "Logging your demise…"). Daarna nog
een 1s `flyTo` + een 1.25s full-screen DropFlash er bovenop. → voelde traag.

**Fix — de card opent op de klik-frame, server bevestigt op de achtergrond:**
- **`src/lib/types.ts`**: nieuw `DropForm` (name/provider/minutes/share/coords/message) — de
  hand-off van de modal naar de page.
- **`src/app/page.tsx`**:
  - `buildOptimisticPin(form, arriveLoc)` — bouwt lokaal een volledige `Pin` (temp id
    `tmp_<uuid>`, client-`obfuscate`'de coords, `recoverAt` = now + minutes). Coords-keten:
    GPS → arrival-loc → `hubForSeed(userId)` (zelfde last-resort als de server).
  - `dropPin(form)` → `openMyCard(optimistic)` (seedt `["pins"]` + `["pin", id]`, zet
    `myPinId`, firet DropFlash, `focusOn({…, snap:true})`) → `setModalOpen(false)` →
    `createPin(...)` op de achtergrond → `.then(reconcileDrop)` / `.catch(rollbackDrop)`.
  - `reconcileDrop(tempId, res)` — wisselt temp→echte pin (echt id/coords/land/vlag) in de
    cache, `setMyPin`/`setMyPinId` = echt id (persist zodat reload de card herstelt),
    re-`focusOn` snap, achievement-toast.
  - `rollbackDrop(tempId, err)` — haalt de temp pin uit de cache, clear `myPinId`, warn-toast
    met de server-reden (409 active / rate / cooldown / moderation).
  - whereami-fetch loopt nu voor **iedereen** bij load (early-return `if (getMyPin()) return`
    weg) zodat ook een resurrected re-dropper accurate coords heeft → geen reconcile-sprong.
- **`src/components/MapView.tsx`**: `FocusTarget.snap?` — in `FocusController` bij snap
  `map.setView(..., {animate:false})` i.p.v. de 1s `flyTo`. Card staat er de frame dat 'ie landt.
- **`src/components/KillModal.tsx`**: nu **pure form** — geen `createPin`/`await`/spinner/inline-error
  meer. `submit()` roept `onDrop(DropForm)` + `onClose()`. Errors komen via toast vanuit de page.

### 2. Premium eigen card — "wow, dit is mijn card"
- **`src/components/PinPopup.tsx`** gesplitst in een router → **`MyStoneCard`** (jouw premium
  versie) + **`VisitorCard`** (die van anderen, bewust compact). Gedeeld: `WaitingBadge`, `Tally`.
  - MyStoneCard: ambient **provider-glow-halo** van bovenaf, pulserende **★ You**-chip in
    provider-kleur (`vk-you-pulse`), **"YOUR STONE"** + naam groot (23px black) met glow,

- **`src/components/DropFlash.tsx`** + CSS: van een 1.25s full-screen skull-curtain → een
  **snelle boom (0.7s)** die van bovenaf (`top-16%`) omhoog wegvliegt en kleiner is. De card is
  de ster, nooit gegate achter een skull. `vk-drop-flash` rijdt nu omhoog + fade.
- **`src/app/globals.css`**: `vk-mine-reveal`, `vk-you-pulse`, aangepaste `vk-drop-flash`/`vk-drop-ring`.

## Beslissingen / notes
- Bewust **optimistic UI met reconcile/rollback** i.p.v. de server-await behouden — dat ís de
  juiste manier om "instant" te bouwen (geen houtje-touwtje). Correctheid blijft: moderatie/
  rate-limits gebeuren nog server-side, alleen de *weergave* is optimistic.
- Server assigned id/jitter blijft leidend → geen client-gekozen id (abuse-vector vermeden);
  prijs is één marker-remount bij reconcile, verborgen onder de (nu snelle) DropFlash.
- Checks: `tsc --noEmit` schoon. Resterende eslint `set-state-in-effect`-meldingen zijn
  pre-existing patronen in ongewijzigde code (mount-effect, IntroCard, KillModal open-effect).

# Sessie 28 jun 2026 — VibeKilled.rip

## Wat is deze sessie gedaan

### 1. Ambient "down devs" — wereldwijde floor (kaart altijd gevuld)
- Doel: bij binnenkomst is de kaart altijd populated met activiteit ("foetelen, maar leuk").
- **`src/lib/ambient.ts`** (nieuw, pure data): ~47 dev-steden wereldwijd met landcode (elk continent), gewogen provider-pool, "last words"-pool, en alle tunables high-level (floor=45, reconcile=25s, feed-kansen, recovery-range 20–200m).
- **`src/lib/store.ts`**: `ensureAmbientPins()` vult de floor bij. Throttle via Redis `SET NX` lock → max 1 reconcile / 25s, rest is no-op. Writes **gepipelined** (~1 round-trip, ook bij volledige fill). Leeftijden **gestaggerd** (backdated kill + variabele recovery) → gevarieerde countdowns + zichtbare churn.
- **`src/app/api/pins/route.ts`**: roept `ensureAmbientPins()` aan vóór `listPins()`. Eerste bezoeker landt al op gevulde kaart.
- **Scheiding**: ambient pins hebben geen `ownerId` + interne `ambient`-vlag (lekt nooit naar client; `toPin` negeert 'm). Tellen mee in zichtbare activiteit (active-count, kaart, feed-trickle, fireworks) maar NIET in echte analytics (kills/leaderboard/admin/cooldown). `/admin` blijft eerlijk.
- Resurrectie-bookkeeping in `listPins` ge-guard met `isAmbient` (geen counter/cooldown/adminLog; feed alleen probabilistisch).

### 1b. Lokale seeding + auto-arrival-zoom (vervolg, zelfde sessie)
- **`ensureLocalAmbientPins(lat,lng,cc)`** in store: 2–4 faked devs binnen ~5–40km (+jitter → ≤~50km) van de bezoeker. Eigen index `vk:ambient:local` + per-regio lock (`vk:ambient:local:<cell>`, 10m) zodat een drukke regio niet opstapelt. Beïnvloedt de globale floor (45) niet.
- **`buildAmbientPin(now, origin?)`** gerefactored om een origin te accepteren; gedeelde `writeAmbientDrafts()` helper (geen duplicatie tussen global/local).
- **`geo.nearbyPoint()`** — willekeurig punt in een km-band rond een origin (random bearing), daarna `obfuscate`.
- **`/api/whereami`** (nieuw, force-dynamic, no-store): geeft coarse self-locatie (afgerond ~11km) terug uit `x-vercel-ip-*` headers én triggert de lokale seeding. Prod-only (lokaal geen headers → null).
- **`MapView.ArriveController`** + `arriveTarget` prop: vliegt eenmalig naar de regio (zoom 5) bij binnenkomst. **`page.tsx`** haalt `fetchWhereami()` op bij mount als je geen eigen pin hebt (anders centreert je eigen pin al).
- Thomas-keuze: radius binnen ~50km (niet 150), 2–4 drops. Gevarieerde wachttijden zat al in `buildAmbientPin` → organisch.

### 1c. Schaal-fixes (na live-test: kaart groeide ongecontroleerd + onhandelbaar)
- **Oorzaak groei**: lokale seeding had geen global cap → veel verschillende bezoeker-regio's (+ crawlers/bots/eigen curls) seedden elk → onbegrensd. **Fix**: `LOCAL_CAP=35` harde ceiling in `ensureLocalAmbientPins` (zcard-check vóór mint).
- **Oorzaak "onhandelbaar"**: `AutoFollowController` vloog naar elke nieuw-geziene pin → met constant nieuwe ambient pins joeg de kaart elke paar sec weg. **Fix**: component **verwijderd** (kaart is nu toch altijd levend).
- **Refresh re-spawnt niet meer**: `takeFirstVisit()` (localStorage `vk:visited`) gate → whereami krijgt alleen `?seed=1` bij een echt nieuw device. Server seedt alleen met die flag; throttle + cap blijven backstop. Geverifieerd op prod: 3× refresh-call → pins blijven gelijk.
- **Animatie-storm bij binnenkomst**: alle markers hadden infinite ping + drop-in tegelijk. **Fix**: `makeIcon` gated op freshness (`FRESH_KILL_MS`/`FRESH_RISE_MS` vs createdAt/recoverAt) → statische halo default, `vk-pin-pulse` korte self-stopping puls (count 2) + `vk-pin-new` alleen verse kills. Backdated ambient = nooit "fresh" → rustig.
- **DOM-schaal**: `cullPins` (viewport-bounds via `BoundsTracker`/`useMapEvents`) + `RENDER_CAP=400`; eigen pin + gefocuste pin altijd behouden. Weather/Fireworks zien nog alle pins.
- **Fireworks-cap**: `MAX_BURSTS_PER_UPDATE=5` + alleen on-screen bursts.
- **Arrival landt op kaartje**: `nearestPin(allPins, loc)` → focus (opent popup met sympathy/Good4U). Fallback `flyToRegion` als nog geen pins.
- Redis-status na fixes (prod): main index 70 (45 floor + 20 local + ~5 echt), local onder cap. Geen handmatige cleanup nodig — self-prune deed het werk.

### 2. Resurrectie-notificaties (toestemming ná de pin-drop)

## Beslissingen / notes
- Permissie via **Enable-knop**, niet auto native popup: na de `await` van de create-call verliest Safari/iOS het user-gesture → onbetrouwbaar/geblokkeerd. Kaartje verschijnt auto ná drop; OS-popup op klik. Alternatief (auto native popup in `onCreated`) is 1 regel als Thomas dat liever wil.
- **Closed-tab limiet**: page-notificatie bereikt background-tab maar niet een gesloten tab. Echt terughalen na sluiten = **Web Push** (service worker + VAPID + server-push bij resurrectie). notify-module is zo opgezet dat die upgrade later inschuift.
